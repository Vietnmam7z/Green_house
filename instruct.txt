Lưu trữ tài khoản bằng cơ sở dữ liệu SQLite. Đọc nội bộ không phải cloud
SQLite đã sử dụng B-Tree để lưu trữ bảng và chỉ mục ( Cây cân bằng)

///////////////////////////////////////////////////////////////////////////////////////////////////////////

id		INTEGER		Khóa chính tự động tăng. Mỗi người dùng có một ID duy nhất.
username	TEXT		Tên đăng nhập của người dùng. Không được để trống.
password	TEXT		Mật khẩu đã được mã hóa bằng SHA-256. Không lưu mật khẩu gốc.
role	TEXT	Vai trò 	của người dùng: "user" hoặc "admin". Dùng để phân quyền.
is_online	INTEGER		Trạng thái kết nối: 1 là đang online, 0 là offline.
last_active	TEXT		Thời điểm cuối cùng người dùng hoạt động (định dạng ISO datetime).

///////////////////////////////////////////////////////////////////////////////////////////////////////////

Hướng dẫn luồng thực hiện đăng nhập web.

[Client] → Gửi username/password
    ↓
[Server] → Kiểm tra DB → Tạo session → Gửi session ID
    ↓
[Client] → Gửi session ID trong các request tiếp theo
    ↓
[Server] → Dùng session ID để xác định người dùng

///////////////////////////////////////////////////////////////////////////////////////////////////////////


Có nên mã hóa mật khẩu ở phía client không?
✅ Có thể, nhưng không bắt buộc:
Một số hệ thống bảo mật cao sẽ mã hóa mật khẩu ngay trên trình duyệt (bằng JavaScript) trước khi gửi.

Tuy nhiên, nếu bạn đã dùng HTTPS, thì dữ liệu gửi đi (bao gồm mật khẩu) đã được mã hóa toàn bộ trong quá trình truyền tải, nên không cần mã hóa thủ công ở client.

///////////////////////////////////////////////////////////////////////////////////////////////////////////

Session token duy nhất
Khi người dùng đăng nhập, hệ thống tạo một session_token duy nhất

Nếu có phiên đăng nhập mới, token cũ bị vô hiệu hóa

Chỉ một phiên hoạt động được phép tồn tại

Code test offline . Dùng hàm đăng nhập và bật DB Browser for SQLite để xem trạng thái. Sau đó theo dõi bằng hàm đăng xuất. Chạy trong file authentication.py

manager = UserManager()
log = UserLogger()
otp = OTPManager(manager)
auth = Authentication(manager, log, otp)
result = auth.offline_user("alice")


Nên nhớ đây là hàm offline không phải đăng xuất. Bản chất hàm đăng xuất vẫn chạy hàm offline này nhưng mục đích chính của offline còn là quan sát người dùng có bị mất kết nối hay thoát web hay không. Thế nên, đây không phải hàm đăng xuất mà nó rộng hơn đăng xuất.

Để tôi ưu vấn đề người dùng rời trang hoặc mất kết nối thì nên dùng JS 2 dạng câu lệnh này (hỏi AI để hiểu thêm):

// Gửi ping định kỳ
setInterval(() => {
    fetch("/ping", { method: "POST", body: JSON.stringify({ username: "alice" }) });
}, 30000);

// Gửi beacon khi rời trang
document.addEventListener("visibilitychange", function () {
    if (document.visibilityState === "hidden") {
        navigator.sendBeacon("/logout", JSON.stringify({ username: "alice" }));
    }
});
